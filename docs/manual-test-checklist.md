# 人工测试清单

以下部分优先使用自动化测试；自动化不稳定或成本过高时，必须执行本清单。

## 1. 导入弹窗

- 打开“导入账号”弹窗，确认默认显示“文本导入”
- 点击“文件上传”，确认页签切换正常
- 再切回“文本导入”，确认文本区域仍可输入
- 拖入 `.txt` 文件，确认文件名显示
- 点击“点击上传”，选择 `.xlsx` 文件，确认文件名更新

预期：

- 页签显示与内容区域一致
- 只接受 `.txt/.xlsx/.xls`
- 文件名区域正确显示当前选择文件

## 2. 文本导入

- 粘贴 `Tab` 分隔账号，执行“追加导入”
- 粘贴 `----` 分隔账号，执行“追加导入”
- 使用“覆盖导入”替换现有账号

预期：

- 表格账号数与导入行数一致
- 分组、Client ID、过期时间正确显示
- 覆盖导入后旧账号被清空

## 3. 文件上传

- 上传 `.txt`
- 上传 `.xlsx`
- 上传包含表头的 Excel

预期：

- 文件能成功导入
- Excel 表头会被忽略
- 导入顺序与文件顺序一致

## 4. 导出

- 不勾选账号直接导出
- 勾选单个账号导出
- 勾选多个账号导出

预期：

- 下载文件名为 `邮箱账号导出_YYYY-MM-DD.txt`
- 内容为 `邮箱[TAB]密码[TAB]ClientID[TAB]刷新令牌`
- 选择导出时只包含选中的账号

## 5. 真实同步

- 使用 [test_mail.txt](/D:/workspace/code/mail/test_mail.txt) 执行“覆盖导入”
- 点击“同步全部”
- 等待批量任务完成，不要在 30 秒内就判定失败；当前默认策略会按账号分批执行，5 个账号大约需要 20 到 30 秒
- 记录每个账号的状态和错误信息
- 至少抽查 1 个同步成功账号，打开邮件预览

预期：

- 页面不会崩溃
- 每个账号都显示明确状态
- 成功账号可看到最近邮件列表
- 失败账号保留明确错误信息
- 若出现 `AADSTS90055`、`throttled`、`timeout` 等可重试错误，再次执行同步时应能在分批节流后恢复成功

## 6. 编辑与删除

- 编辑一个账号的分组
- 删除一个账号

预期：

- 编辑后列表立即更新
- 删除后账号和对应邮件记录一起消失
